# 上线后验收清单（AI 学习站）

> 每次上线（main 分支合并 / GitHub Pages 部署）后按本清单逐项核验。  
> 完成后在 PR 或发版记录中附上日期、执行人和截图/日志。

## 0. 发版确认
- [ ] `main` 分支与 `origin/main` 同步完成，`git status` 为 clean。
- [ ] 本次改动已经推送到 GitHub，Actions 有新的 `Deploy GitHub Pages` 运行记录。
- [ ] Workflow 运行通过（绿色）：`Validate static site`、`Deploy` 均成功。
- [ ] Pages `url` 与预期站点一致（建议为：`https://happywa147.github.io/ai-learning-site/`）。
- [ ] 清空浏览器缓存后首次访问站点正常，不出现 404/502。

## 1. 基础页面与路由
- [ ] 首页（`#home`）加载正常，主标题、按钮、快速入口可见。
- [ ] 下列锚点页签可正常跳转且内容完整：
  - `#intro`、`#models`、`#toolkit`、`#resource-radar`、`#projects`、`#weekly`、`#feedback`
- [ ] `index.html`、`styles.css`、`app.js`、`sitemap.xml`、`robots.txt`、`llms.txt`、`LICENSE` 在线上可访问。
- [ ] 无明显控制台报错（首屏加载 3 次内无未捕获错误）。

## 2. 核心功能验收（功能性）
- [ ] 新手上手流程可完成（推荐时长：30 秒）
- [ ] 签到按钮、XP/等级、徽章逻辑可见可用。
- [ ] 作品点亮与周任务记录可正常更新并持久化。
- [ ] 月更区可正常显示最新月度、卡片切换、状态/摘要/任务/避坑。
- [ ] AI 项目挑战库可搜索、筛选、复制挑战模板。
- [ ] Agent 角色学院可复制角色卡，且“今日 Agent 挑战”可生成内容。
- [ ] 资源雷达支持“处理建议/许可证/搜索”筛选，条目计数显示准确。
- [ ] 注册草稿提交：
  - 姓名/微信号/目标路径必填校验通过；
  - 草稿提交后可见本地落库提示；
  - 联系内容可复制（邮箱不直接明文展示）。
- [ ] 赞助窗口二维码可打开/可见，不会与微信联系二维码混淆。
- [ ] 反馈功能：CSV 导出、Issue 草稿、周报导出均可触发。

3. 内容与合规
- [ ] 月更 JSON `status` 字段：
  - 已发布条目为 `published`
  - 草稿条目为 `draft` 且不会默认展示
- [ ] 资源/角色/项目条目在 UI 与 `app.js`/数据源一致（无空白字段显示异常）。
- [ ] 无敏感信息在页面明文外露（邮箱/二维码仅按约定展示方式呈现）。
- [ ] 第三方链接均有 `target="_blank" rel="noopener noreferrer"`（如有外链）。
- [ ] 月更内容至少包含可复核来源链或可验证任务。

## 4. 体验与可用性
- [ ] 390x844 与 1600x900 关键页面无重叠、溢出与明显抖动。
- [ ] 按钮/筛选/卡片 hover 与点击反馈明显。
- [ ] 首屏加载后 3 秒内无白屏时间过长现象。
- [ ] 关键文本可读（颜色对比不低于基本可读水平，CTA 按钮可识别）。
- [ ] 导航与筛选在移动端仍可操作（至少可完成一次主流程）。

## 5. 数据与稳定性
- [ ] 本地 `localStorage` 有效：刷新后签到、积分、任务、周报草稿保留或可重置。
- [ ] 删除/重置相关操作不会误清除历史数据。
- [ ] 页面无明显内存增长迹象（重复操作后仍能流畅交互）。

## 6. 事故处理与回滚
- [ ] 发现红线问题（白屏、核心功能不可用、部署 404）立即冻结继续发布。
- [ ] 若需回滚，回滚到上一个可用提交并更新 Issue/群组状态。
- [ ] 在 `Issues`/`README` 中记录本次异常与复盘要点。

## 7. 发布后 24 小时巡检
- [ ] 再次访问首页验证站点可达。
- [ ] 验证核心路径中任一入口都能进入月更区和资源雷达。
- [ ] 检查 GitHub Pages 访问量和主要交互反馈（若有）
- [ ] 至少补一条用户反馈或任务建议记录，证明反馈链路通畅。
