# 路线图（ROADMAP）

这个项目的长期目标，是成为一个持续更新、贴近真实应用市场的 AI 学习工具，而不是一次性课程页面。

## 当前定位

AI 原生能力自学站 = 学习路线 + 游戏化任务 + 月度更新 + 作品集 + 反馈循环。

它要解决的问题：

- AI 变化太快，固定教材很容易过时。
- 新人不知道从哪里开始。
- 学习者容易只收藏工具，不形成作品。
- 国内外模型、工具和应用市场变化需要持续复盘。

## 近期目标（1-2 周）

- 完成 GitHub Pages 稳定部署与 README 门面优化。
- 补齐截图、项目封面和传播素材说明。
- 完善 60 秒新手上手区，让首次访问者能快速开始。
- 优化移动端导航和关键区块排版。
- 增加每月更新模板，让非开发者也能贡献内容。

## 中期目标（1-2 个月）

- 建立更完整的月度更新机制：
  - 模型变化
  - 中国应用市场
  - 短视频 / 内容生产
  - AI 编程 / Agent / Skill
  - 风险、合规和事实核查
- 增加更多作品样例：
  - AI 学习助教报告
  - 短视频脚本工厂
  - 个人知识库问答
  - Agent 小工具
  - AI 产品观察报告
- 支持更多导出能力：
  - 学习周报
  - 作品集摘要
  - 反馈 CSV
  - 月度内容清单

## 长期目标（3-6 个月）

- 把网站发展成可复用的 AI 学习系统模板。
- 形成面向不同人群的学习路径：
  - 零基础
  - 职场人
  - 内容创作者
  - 开发者
  - 业务负责人
- 建立公开贡献机制，让社区共同维护月度更新。
- 支持更专业的数据层：
  - 模型对比数据独立成 JSON
  - 项目样例独立成 JSON
  - 月更内容可版本化追踪
- 探索后端或表单服务接入，但保持静态站可独立运行。

## 优先级规则

P0：影响核心可用性的问题，例如页面打不开、任务无法保存、月更无法展示。  
P1：影响高频学习路径的问题，例如签到、周报导出、路线选择、作品点亮。  
P2：影响体验和传播的问题，例如移动端拥挤、README 不清楚、截图缺失。  
P3：打磨项，例如文案润色、动画节奏、更多样例和更多模板。
