## 五、小程序营销策略

### 5.1 小程序独有优势

| 优势                 | 营销利用方式                   |
| -------------------- | ------------------------------ |
| 无需下载，扫码即用   | 网站放置 QR 码，引导小程序体验 |
| 微信生态内分享       | 分享卡片带网站品牌，裂变传播   |
| 签到提醒（订阅消息） | 提高次日留存（需用户授权）     |
| 与网站数据互通       | 多端学习进度同步（长期目标）   |

### 5.2 小程序上线推广步骤

#### 阶段一：提审准备（上线前 1 周）

- [ ] 替换 `project.config.json` 中的 AppID
- [ ] 准备小程序截图（5 张，含首页、路线、Agent 卡、个人中心）
- [ ] 撰写小程序简介（80 字以内）
- [ ] 选择服务类目：「教育 > 在线教育」
- [ ] 准备隐私协议（已有一份，需补充小程序条款）

#### 阶段二：审核期间（预计 3-7 天）

- 同步在网站首页添加「小程序即将上线」的横幅
- 准备首批种子用户名单（邀请码预发）

#### 阶段三：审核通过后（上线当天）

1. **公告发布**：在知乎、小红书、GitHub Discussions 同步发布上线公告
2. **种子用户激活**：向预注册的 50 名用户发送小程序码
3. **微信好友分享**：发动个人关系网，每人至少分享到 3 个群

### 5.3 小程序裂变机制设计

> 利用已有的邀请码系统，设计小程序专属裂变活动

#### 活动：「邀请 3 人，解锁专属 Agent 卡」

```
用户 A 生成专属邀请码 → 分享给 B/C/D
B/C/D 通过邀请码注册并完成首次签到
→ A 获得 500 XP + 专属徽章「传播者」
→ B/C/D 各获得 100 XP 新人奖励
```

#### 小程序分享卡片优化

当前分享卡片内容（建议在 `pages/index/index.js` 的 `onShareAppMessage` 中优化）：

```javascript
onShareAppMessage() {

    imageUrl: '/assets/images/share-card.png' // 需设计专用分享图
  }

```

## 十二、总结

本营销计划以**零预算启动、内容驱动增长**为核心策略，充分利用：

1. **知乎 + 小红书** 作为流量入口
2. **GitHub 开源** 作为信任背书
3. **游戏化机制** 作为留存引擎
4. **月更内容** 作为持续关注理由
5. **小程序 + 网站双端** 覆盖不同场景

前 3 个月聚焦**冷启动和种子用户**，第 4-6 个月转向**增长和社区运营**。所有动作均可由个人独立完成，无需团队支持。
