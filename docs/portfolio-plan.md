## 二、三条线，三种逻辑

你的 5 个项目天然分成三条线，每条线的目标、投入和回报逻辑完全不同。**不要混在一起算账。**

### 教育线 — 给儿子的 18 岁礼物

**包含**：ai-learning-site + handbooks-zh

**核心定位**：这不是生意，是知识传承。你给儿子做的 AI 自学体系，顺便开放给所有人用。

**成功标准**：

- 儿子能用它学会 AI 基础（而不是你赚了多少钱）
- 月更不断，12 个月内积累 50-100 个真实用户
- 小程序上线，移动端可用

**投入上限**：每月 5-8 小时（月更 + 偶尔维护），**不搞直播、不搞训练营、不搞社群运营**

**变现逻辑**：不追求直接变现。如果有人主动找上门要付费，再考虑。否则就是免费开源项目。

### 产品线 — 实体生意

**包含**：aerotether-site（AEROTETHER 眼镜绑带）

**核心定位**：实体产品出海/内销，落地页做 SEO + waitlist 验证需求。

**成功标准**：

- waitlist 收集 100+ 邮箱
- Google 搜索「eyewear tether」「glasses strap」等关键词有排名
- 产生第一笔真实订单

**投入上限**：产品开发另算（不在此规划范围内），网站维护每月 1-2 小时

**变现逻辑**：直接卖货。这是唯一一条「做生意」的线。

### 品牌线 — 个人 IP

**包含**：GitHub 全部仓库作为作品集 + 未来可能的内容输出

**核心定位**：你的 GitHub 主页就是你最好的简历。5 个项目摆在那里，比任何自我介绍都有说服力。

**成功标准**：

- GitHub Profile 优化（bio + pinned repos + 自我介绍 README）
- 在知乎/小红书发 3-5 篇「我用 AI 在 26 天内做了 5 个项目」的复盘文章
- 有人因为看到你的作品，主动找你合作/咨询/给工作机会

**投入上限**：一次性 4-6 小时优化 Profile + 写 3 篇文章，之后零维护

**变现逻辑**：间接变现 — 更好的工作机会、咨询邀约、人脉资源

## 四、每个项目的具体策略

### 1. AI 练级站（ai-learning-site）

**定位**：给儿子的礼物 + 开源作品。不是生意。

**做什么**：

- 月更学习路线（每月 2-3 小时）
- 小程序上线后就不管了，自然搜索
- 不搞社群、不搞直播、不搞训练营
- 如果有人 Star/Fork/Issue，及时回复

**不做什么**：

- 不花钱投流
- 不主动找 KOL 推广
- 不做付费功能

**成功标志**：儿子 18 岁生日时（确认日期），网站 + 小程序都在线，能看到他在用。

### 2. 中文使用手册合集（handbooks-zh）

**定位**：最有变现潜力的内容资产。20 万字、9.5/10 评分、6 本完整手册——这是现成的知识付费产品。

**做什么**：

- GitHub 上保持免费开源（引流）
- 知乎/小红书发精华摘录（引流）
- 考虑做成 PDF/电子书在知识星球/公众号分发（变现路径）
- **不主动做付费，等有人问「有没有完整版/PDF 版」时再开」

**变现可能**：

- 知识星球 ¥49-99/年，放手册 PDF + 答疑
- 公众号连载，积累关注后接广告
- 企业团购（给团队做 AI 工具培训材料）

**关键**：内容已经做好了，零边际成本。有 1 个人买就赚，有 100 个人买就大赚。

### 3. AEROTETHER（aerotether-site）

**定位**：实体产品的线上门面。

**做什么**：

- 提交 Google Search Console / Bing Webmaster
- 等 waitlist 积累 50+ 邮箱后，发第一封触达邮件
- 如果有 TikTok/小红书运营计划，落地页作为流量承接页

**不做什么**：

- 在没有真实产品照片/测评数据前，不投流
- 不做内容营销（产品线不靠内容，靠产品本身）

### 4. AI 架构师手册（ai-architect-handbook-code）

**定位**：空仓库，6 月 8 日建的占位。

**建议**：

- 如果短期没有计划，删掉或设为 private（避免给人「烂尾」印象）
- 如果想做，排在手册合集和 AI 练级站之后，优先级最低
- 可以等儿子学完 AI 练级站基础后，作为「进阶篇」一起做

### 5. GitHub 个人主页

**定位**：你所有项目的入口，最好的个人名片。

**做什么**（一次性，2 小时）：

- 写一个 `happywa147/happywa147` 仓库的 README.md（会显示在 Profile 主页）
- 内容：一句话介绍 + 5 个项目卡片 + 联系方式
- Pin 4 个最有代表性的仓库（ai-learning-site, handbooks-zh, aerotether-site, github-handbook-zh）
- 加 topics 标签：`ai-education`, `chinese`, `open-source`, `claude`, `cursor`

## 五、关于「给儿子」这件事

你说这个 AI 自学站是给儿子 18 岁高中毕业定的必须跟上的知识。这个出发点比任何商业计划都重要。

### 你已经给了他什么

1. **一套完整的 AI 学习体系** — 不是零散的视频，是有路线、有进度、有考核的系统
2. **67 个 AI Agent 角色卡** — 他可以直接拿来用，理解 AI 能做什么
3. **6 本工具手册** — GitHub、ChatGPT、Claude、Gemini、Cursor、Claude Code，从入门到进阶
4. **一个父亲亲手做的网站** — 这本身就是最好的示范：「AI 时代，一个人可以做什么」

### 你还可以给他什么

| 礼物                | 怎么做                                         | 时机         |
| ------------------- | ---------------------------------------------- | ------------ |
| 一个 GitHub 账号    | 帮他注册，把 AI 练级站 fork 给他               | 18 岁生日前  |
| 第一个 Pull Request | 让他给 AI 练级站提一个 PR（哪怕改个错别字）    | 生日当天     |
| 手册打印版          | 把 6 本手册打印成纸质版，作为实体礼物          | 生日当天     |
| 「共同开发者」身份  | README 里加一行：「Co-developed with my son」  | 他开始学之后 |
| 一个真问题          | 让他用 AI 解决一个你的真实问题，把过程记录下来 | 他学完基础后 |

### 教育目标

不是让他「学会用 ChatGPT」，而是让他理解：

1. **AI 是工具，不是答案** — 工具的价值在于使用者的判断力
2. **动手做 > 看教程** — 签到/XP/徽章设计的初衷就是让他动手
3. **开源精神** — 他用的东西是免费的，他也可以创造免费的东西给别人用
4. **持续学习** — 月更的意义：知识不是一次性的，是持续的

**这个网站的真正 KPI 不是 UV/DAU，是你儿子有没有在用。**

## 六、关于「期待不大」

你说期待不大。让我重新算一笔账：

### 之前的算法（把 AI 练级站当生意）

```
投入：每月 40-60 小时 × 12 个月 = 480-720 小时
收入：第 1 年 0-3 万
时薪：¥40-60/小时
结论：不如上班
```

### 现在的算法（三条线分开算）

**教育线**：

```
投入：每月 5 小时 × 12 个月 = 60 小时
收入：¥0
回报：儿子学会 AI + 一个开源作品
时薪：无法用钱衡量
```

**品牌线**：

```
投入：一次性 6 小时 + 每月 2 小时 = 30 小时/年
收入：¥0 直接收入
回报：简历加分 + 可能的合作/工作机会
时薪：无法用钱衡量（但可能比教育线更值钱）
```

**产品线**（AEROTETHER）：

```
投入：每月 1-2 小时（仅网站维护）
收入：取决于产品本身，不在此规划范围
回报：实体生意
```

**手册合集**（被动收入）：

```
投入：已完成（0 边际成本）
收入：如果开知识星球，¥49-99/人/年
      100 人 = ¥5000-10000/年（纯被动）
回报：内容已经做好了，有人买就是纯赚
```

### 真实结论

```
总投入：每月 8-10 小时
直接收入：0-1 万/年（手册合集可能产生）
间接回报：儿子教育 + 个人品牌 + 可能的机会
```

**这不是一个「期待不大」的生意，这是一个「几乎零成本、零风险、回报多元」的个人项目组合。**

你不需要期待它赚多少钱。你需要期待的是：

1. 儿子在用
2. GitHub 上有人 Star
3. 偶尔有人因为看到你的作品来找你聊

这三件事任何一件发生了，你的 30 小时/年就没有白花。
