# 可上线前最终检查清单（动作回归版）

版本目标：在 `git push origin main` 后触发 GitHub Pages 部署的当天完成最终验收。

## 一、快速执行（先做）

1. **环境与路由（P1）**
   - `main` 内容为最新：`git status` 干净，`origin/main` 包含最新 commit。
   - `?page=home` / `?page=starter` / `?page=map` / `?page=worldview` / `?page=game` / `?page=models` / `?page=resources` / `?page=showcase` / `?page=projects` / `?page=agents` / `?page=weekly` / `?page=monthly` / `?page=toolkit` / `?page=opensource` / `?page=faq` / `?page=feedback` / `?page=register` 都可直接访问且仅显示单页内容。
   - 先后打开 390x844 和 428x926：导航、分页与主内容不出现水平溢出。

2. **动作回归（P1）—按下列路径逐项确认**
   - 签到：点击“今日签到”，XP+20，按钮文案变“今日已签到”。
   - 任务完成：在“周计划”打勾任一周，需先填写证据，取消勾选后可回滚。
   - 作品点亮：项目页点击“点亮作品 +80 XP”，再次点击可取消点亮。
   - 复制动作：在资源雷达、项目挑战、Agent 角色三处任意一项执行复制按钮，成功文案应出现。
   - 分页跳转：从首页依次点击导航到每个栏目，`#game`、`#resources`、`#register` 等都能正常切到并返回。
   - 周报导出：点击“导出学习周报”可触发并下载 `.md`。

3. **边缘区块可达性（P2）**
   - 赞助弹窗可打开、Esc 与遮罩关闭。
   - 注册表单、反馈表单可在移动端滚动到操作按钮。
   - 证据弹窗（周任务/作品）支持 Esc、点击遮罩、提交/取消。

4. **移动端体验节奏（P2）**
   - 手机端按钮高度最小 44px，列表间距一致，首屏无“点不到/挤一起”。
   - 允许垂直滚动查看长内容，顶部返回/切页不丢焦点。
   - 连续 2 次执行“分页跳转”测试后，无明显白屏停顿。

5. **可复制动作与数据安全（P1）**
   - 邮箱不直接明文全量展示；联系稿与复制逻辑可正常使用。
   - 本机 `localStorage` 持久化：签到/周任务/作品/反馈/报名有更新；`清空`有二次确认。

## 二、验收结果打分（建议）

- **P0（阻塞）**：无法打开页面 / 核心交互 500 错误 / 导航不可用。
- **P1（高优先）**：签到、分页跳转、周报导出、复制按钮、任务点亮中任一不可用。
- **P2（中优先）**：移动端按钮密度不一致、列表间距偏离、滚动到达性偶发卡顿。
- **P3（优化）**：文案精修、动画节奏、统计表述和视觉统一。

评分建议：
- 90 分以上：可继续对外发布。
- 80–89 分：仅允许预发布，待修复 P1 后再公开。
- 80 分以下：先暂停发布，集中修复 P1。

## 三、可运行的回归（代码内置）

在控制台执行：

```js
__aiLearningActionRegression()
```

返回字段示例：
- `passed`：是否全部通过
- `checks`：签到/任务/导出/复制/分页的单项核验
- `pageJumpChecks`：每个页面跳转的可见性结果

也可执行最终评分入口：

```js
__aiLearningPreLaunchReadiness()
```

返回字段示例：
- `score`：0-100 分，按 P0/P1/P2 阻塞权重计算
- `level`：`可发布` / `预发布（待修）` / `阻塞中`
- `passed`：P0 且 P1 无阻塞
- `p0Failed` / `p1Failed` / `p2Failed`：分级失败项数量
- `checks`：带分类的打分清单
